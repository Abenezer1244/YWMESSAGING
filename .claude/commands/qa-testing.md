---
name: qa-testing
description: Test planning, test case creation, bug reporting, and quality assurance
---

Use the QA Testing agent to plan tests, create test cases, and validate feature quality.

Specify the feature or area to test, and the agent will:
1. Create comprehensive test plan
2. Design test cases with edge cases
3. Plan automation opportunities
4. Identify quality risks
5. Provide bug report templates

Example usage:
```
/qa-testing

Create a test plan for the 10DLC delivery tier feature including acceptance criteria and test cases.
```
